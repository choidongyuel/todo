# 요구사항 문서 (01-requirements.md)

## 프로젝트 개요

**앱 이름:** 일정관리 앱 (Todo App)  
**핵심 목적:** 매일 할 일을 개인적으로 관리하고, 카테고리·우선순위로 빠르게 분류할 수 있는 반응형 웹앱  
**주 사용자:** 개인 사용자 (나만 쓸 거)  
**주 사용 디바이스:** 데스크톱 + 모바일 (반응형)  
**주 사용 시점:** 모바일에서도 자주 추가/체크 (언제든 빠른 접근 가능)

---

## 기술 스택

| 분류 | 선택 |
|------|------|
| 프론트엔드 | React + TypeScript |
| 스타일링 | CSS-in-JS 또는 CSS Modules (결정 보류, 아키텍처 단계에서) |
| 상태 관리 | useState (간단한 스코프) |
| 데이터 저장 | localStorage (브라우저 로컬) |
| 빌드 도구 | Vite (추천) 또는 Create React App |

## 데이터 모델

### Todo 항목 (Task)

```typescript
interface Task {
  id: string;              // UUID
  title: string;           // 할 일 제목
  category: string;        // 카테고리 (예: "일", "개인", "쇼핑" 등)
  priority: 'low' | 'medium' | 'high';  // 우선순위
  completed: boolean;      // 완료 여부
  createdAt: number;       // 생성 시간 (timestamp)
  completedAt?: number;    // 완료 시간 (선택)
}
```

### App 상태

```typescript
interface AppState {
  tasks: Task[];
  filter: 'all' | 'active' | 'completed';
  sortBy: 'priority' | 'createdDate' | 'category';
  selectedCategories: string[];  // 필터링용 선택된 카테고리
}
```

## 핵심 기능 요구사항

### 1. 할 일 관리 (CRUD)

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| **생성** | 제목 + 카테고리 + 우선순위 입력 후 추가 | P0 |
| **조회** | 모든 할 일을 리스트로 표시 | P0 |
| **완료 마크** | 할 일을 완료로 표시/해제 (체크박스) | P0 |
| **삭제** | 할 일 제거 | P0 |
| **수정** | 제목, 카테고리, 우선순위 수정 | P1 |

### 2. 필터링 & 정렬

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| **상태 필터** | "전체" / "미완료" / "완료" 탭 | P0 |
| **카테고리 필터** | 선택된 카테고리만 표시 | P1 |
| **우선순위 정렬** | 높음 → 중간 → 낮음 순서로 정렬 (기본) | P1 |
| **날짜 정렬** | 최신순 / 오래된순 정렬 | P2 |

### 3. 데이터 영속성

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| **localStorage 저장** | 브라우저 종료 후에도 데이터 유지 | P0 |
| **자동 저장** | 변경 시 자동으로 저장 | P0 |

## 비기능 요구사항

| 요구사항 | 목표 |
|---------|------|
| **반응형** | 모바일(320px+) / 태블릿 / 데스크톱에서 모두 사용 가능 |
| **접근성** | 키보드 네비게이션 지원, 스크린 리더 기본 지원 |
| **성능** | 100개 항목 이상 로드 시 < 1초 |
| **브라우저 호환** | 최신 Chrome, Firefox, Safari, Edge |

## 범위 제외 (Out of Scope)

반나절(2~3시간) 예산을 고려하여 다음은 **v2 이후**로 미룹니다:

- ❌ 달력 뷰 (리스트 뷰만)
- ❌ 마감일/시간 설정
- ❌ 반복 기능 (매일/매주)
- ❌ 다중 사용자 동기화 / 클라우드 연동
- ❌ 알림 기능
- ❌ 태그 (현재는 카테고리만)
- ❌ 첨부파일, 메모

## 완료 기준 (Definition of Done)

### MVP (Milestone 1~3)
- [ ] 할 일 추가/삭제 동작
- [ ] 완료 체크박스 동작
- [ ] 상태 필터 (전체/미완료/완료) 동작
- [ ] localStorage 저장 및 복원 동작
- [ ] 반응형 레이아웃 (모바일 320px+에서도 정상)
- [ ] 초기 입력 포커싱 (UX 개선)

### v1 Complete (Milestone 4~5)
- [ ] 우선순위 필터링 및 정렬
- [ ] 카테고리 필터링
- [ ] 수정 기능
- [ ] 스타일 다듬기 (색상, 타이포, 여백)

### v1 Polish (Milestone 6~8, 시간 남을 때)
- [ ] 엣지 케이스 처리 (긴 제목, 한글 IME)
- [ ] 라이트/다크 모드 (선택)
- [ ] 성능 최적화

## 사용자 여정 (Happy Path)

1. 앱 열기 → 기본 화면 (빈 리스트)
2. "할 일 추가" 버튼 → 입력 필드 활성화
3. 제목 + 카테고리 + 우선순위 입력 → "추가" 클릭
4. 리스트에 새 항목 표시
5. 완료하면 체크박스 클릭 → 항목 흐린 표시
6. 필터 "완료됨" 탭 클릭 → 완료 항목만 보기
7. 삭제 버튼 클릭 → 항목 제거
8. 브라우저 종료 후 재방문 → 모든 데이터 복원
