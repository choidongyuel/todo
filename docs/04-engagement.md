# 차별화·성취감 문서 (04-engagement.md)

## 앱 컨셉: "빠르고 만족스러운 할 일 관리"

사용자가 이 앱을 계속 쓰게 만드는 핵심은 **빠른 입력 + 명확한 우선순위 + 완료의 만족감**입니다.  
복잡한 기능 대신 **가장 중요한 순간들(추가, 우선순위 변경, 완료)**에서 좋은 경험을 제공합니다.

---

## 차별화 기능 (Engagement Features)

### E1: 완료할 때 시각적 피드백

**설명**  
할 일을 완료할 때(체크박스 클릭) **즉시 시각적 보상**을 제공하여 심리적 만족감을 극대화합니다.

**구체적 구현**
- **체크박스 클릭 순간:** 작은 폭죽 이모지(🎉) 또는 체크마크 애니메이션 표시
- **색상 변화:** 항목이 밝은 초록색(완료)으로 순간적으로 변했다가 흐린 회색으로 안정화
- **위치 변화(선택):** 완료 항목을 리스트 하단으로 부드럽게 슬라이드하거나, "완료됨" 섹션으로 이동
- **소리(선택):** 매우 작은 "핑(ping)" 소리 또는 무음

**사용자 경험**  
→ 민준: "메일에서 할 일 추가 후 완료할 때 즉시 만족감 느낌"  
→ 지은: "저녁에 하루 정리할 때 각 항목마다 작은 성취감 누적"

**개발 복잡도:** 낮음 (CSS 애니메이션 + 간단한 애니메이션 라이브러리)

---

### E2: 우선순위 자동 정렬

**설명**  
할 일의 우선순위를 변경하면 **즉시 리스트가 재정렬**되어, 사용자가 "지금 가장 중요한 일이 뭔지" 한눈에 파악할 수 있습니다.

**구체적 구현**
- **정렬 기준:** 높음 → 중간 → 낮음 순서로 자동 정렬
- **같은 우선순위끼리:** 추가된 순서(최신순) 유지
- **색상 구분:** 높음(빨강/주황) > 중간(노랑) > 낮음(회색) 시각적 구분
- **재정렬 애니메이션:** 우선순위 변경 시 리스트 항목이 부드럽게 이동

**사용자 경험**  
→ 민준: "긴급 요청 들어왔을 때 우선순위 변경하면 즉시 상위로 올라옴 → 집중력 높아짐"  
→ 지은: "시험까지 남은 시간에 따라 우선순위 자동 조정하면 리스트가 새로 정렬되어 명확함"

**개발 복잡도:** 중간 (정렬 로직 + 애니메이션)

---

### E3: 카테고리별 진도 바

**설명**  
각 카테고리별로 **완료율을 프로그래스 바로 표시**하여, 사용자가 영역별 진도를 한눈에 파악하고 동기를 유지할 수 있습니다.

**구체적 구현**
- **진도 바 위치:** 화면 상단 또는 사이드바에 카테고리별 바 표시
  - 예: "업무: ████░░░░ 60%", "개인: ██████░░ 75%"
- **계산 방식:** 완료한 항목 수 / 전체 항목 수 (미완료 기준)
- **색상:** 카테고리별로 다른 색상 사용 (시각적 구분)
- **인터랙션:** 진도 바를 탭하면 해당 카테고리 필터링

**사용자 경험**  
→ 민준: "업무 카테고리 진도가 70%에 가까워지면서 마지막 스퍼트 동기 부여"  
→ 지은: "각 과목 진도를 한눈에 보면서 '○○학은 아직 40%네' 하고 계획 조정"

**개발 복잡도:** 중간 (계산 로직 + UI 렌더링)

---

### E4: 빠른 추가 (기본값)

**설명**  
**기본값(카테고리="일반", 우선순위="중")**으로 설정하여, 제목만 입력하고 Enter를 누르면 즉시 추가됩니다.  
상세 정보는 나중에 수정 가능합니다.

**구체적 구현**
- **입력 필드가 화면 최상단에 배치**
- **Enter 키:** 제목만으로 즉시 추가
- **선택적 상세 입력:** 카테고리와 우선순위는 드롭다운(기본값 유지)
- **포커스 유지:** 추가 후 입력 필드가 자동으로 다시 포커스됨

**사용자 경험**  
→ 민준: "메일에서 할 일 3개를 15초 만에 추가 완료"  
→ 지은: "공지 받은 과제를 즉시 추가했다가 나중에 카테고리 정리"

**개발 복잡도:** 낮음 (UX 디자인 + 기본값 설정)

---

### E5: 미완료 항목 알림

**설명**  
매일 저녁(설정 가능한 시간) **미완료 항목의 개수를 알림**으로 제공하여, 사용자가 하루를 마무리하기 전에 해야 할 일을 상기하고 미룬 일을 처리하도록 부드럽게 강제합니다.

**구체적 구현**
- **알림 시간:** 기본값 오후 7시(사용자 설정 가능)
- **알림 내용:** "오늘 3개 아직 남았어요 ✌️" 또는 "오늘의 할 일 정리할 시간!"
- **알림 클릭:** 앱으로 바로 이동
- **비침투성:** 브라우저 알림(조용한 수준)

**사용자 경험**  
→ 민준: "저녁 알림으로 잊던 긴급 일감을 생각나게 됨"  
→ 지은: "저녁 7시 알림 보고 오늘 과제 2개를 빠르게 정리"

**개발 복잡도:** 중간 (브라우저 Notification API, 시간 설정)

## 의도적으로 넣지 않는 것 (v2 이후)

반나절 예산과 MVP 범위를 고려하여 **다음 기능들은 v2 이후로 미룹니다:**

| 기능 | 이유 | 예상 시점 |
|------|------|---------|
| 일일/주간 통계 (완료 개수) | 계산 로직 + 차트 라이브러리 필요 | v1.1 |
| 연속 완료 스트릭 | 매일 카운팅 로직 필요 | v1.1 |
| "오늘의 추천" (AI 추천) | 추천 알고리즘 필요 | v2 |
| 주간 완료율 그래프 | 데이터 시각화 라이브러리 | v2 |
| 완료한 것 "회상" 보기 | 필터링/검색 고도화 | v2 |
| 완료할 때 보상음 | 사용자 피드백 후 추가 | v1.1 |
| 일일 목표 설정 (5개 완료하기) | 진행률 트래킹 로직 | v2 |
| 팀 협업 기능 | 원격 DB + 실시간 동기화 | v3 |
| 마감일 설정 | 캘린더 뷰 필요 | v2 |
| 반복 기능 (매일/매주) | 일정 생성 로직 | v2 |

**선택 기준:** MVP는 "**빠르고 깔끔한 추가 + 명확한 우선순위 + 완료의 만족감**" 세 가지에 집중합니다.

## 차별화 요약

| 축 | 이 앱의 전략 | 경쟁사와의 차이 |
|----|-----------|-------------|
| **속도** | 입력 필드 상단 배치 + 기본값으로 3초 추가 | Google Tasks도 빠르지만, 우리는 기본값 최적화 |
| **명확함** | 우선순위 자동 정렬 + 카테고리 진도 바 | Todoist는 복잡, 우리는 '지금 뭐 할지'만 명확 |
| **만족감** | 완료할 때 시각적 피드백 | 대부분 체크만 하고 넘어감, 우리는 축하하는 느낌 |
| **꾸준함** | 저녁 알림으로 미완료 상기 | 강제하지 않으면서 부드럽게 이끌기 |

**결론:** "복잡함을 덜어내고, 가장 중요한 순간(추가·우선순위·완료)에 좋은 경험을 집중"
