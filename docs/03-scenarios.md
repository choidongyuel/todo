# 시나리오 문서 (03-scenarios.md)

## 민준(직장인)의 시나리오

### S1: "메일 폭주 속 빠른 할 일 추가"

**Context**  
아침 9시, 메일을 확인하니 팀원과 팀장으로부터 여러 요청 메일이 들어있다.  
각각을 할 일로 변환해서 빠르게 추가해야 한다.

**Flow**
```
1. 휴대폰으로 메일 앱 열기 → 첫 번째 요청 읽기
2. 메일 앱에서 Todo 앱으로 전환
3. Todo 앱의 입력 필드에 제목만 입력 (예: "보고서 초안 작성")
4. Enter 키 → 즉시 추가됨
5. 다음 메일로 돌아가기 (앱 닫을 필요 없음)
6. 같은 방식으로 3개 추가
7. 몇 분 후 입력한 내용 확인 (우선순위 조정 필요하면 그때)
```

**인사이트**
- 입력 필드가 가장 눈에 띄는 곳에 있어야 함
- **기본값**으로 바로 추가되어야 함 (카테고리와 우선순위는 나중에)
- 앱을 최소한으로 다루고 빨리 메일로 돌아갈 수 있어야 함

---

### S2: "회의 중 새 일감 기록"

**Context**  
팀 회의 중(오전 10시). 팀장이 각자에게 새로운 프로젝트 일감을 배정했다.  
회의를 중단하지 않으면서 빠르게 기록해야 한다.

**Flow**
```
1. 회의실에서 휴대폰으로 빠르게 Todo 앱 열기
2. 입력 필드에 제목 입력 (예: "신규 캠페인 기획서 작성")
3. 우선순위를 "높음"으로 변경 (카테고리는 기본값)
4. Enter → 추가 완료
5. 회의를 계속 진행
6. 회의 끝난 후 카테고리 추가 수정 (필요하면)
```

**인사이트**
- 입력과 우선순위 설정이 3초 이내 완료되어야 함
- 회의 흐름을 끊지 않을 정도로 간단해야 함
- 나머지 정보는 나중에 보충해도 되도록 설계

---

### S3: "긴급 요청에 우선순위 상향"

**Context**  
오후 3시, 팀장에게서 갑자기 긴급 요청이 들어왔다.  
이미 리스트에 있던 일의 우선순위를 급히 높여야 한다.

**Flow**
```
1. 메시지 앱에서 긴급 요청 받음
2. Todo 앱 열기
3. 해당 할 일을 찾음 (예: "데이터 분석 완료")
4. 우선순위를 "낮음"에서 "높음"으로 변경
5. 리스트가 재정렬되어 상위로 올라옴
6. 한눈에 "지금 가장 중요한 일"이 보임
```

**인사이트**
- 우선순위 변경이 **즉시 시각적으로 반영**되어야 함
- 리스트가 자동으로 재정렬되므로 "뭘 먼저 할지" 명확함
- 색상이나 배치로 높은 우선순위를 시각적으로 강조

---

### S4: "점심 시간에 오전 진행 상황 체크"

**Context**  
점심시간(오후 12시). 모바일로 빠르게 오전 진행 상황을 확인하고,  
오후에 할 일을 파악하고 싶다.

**Flow**
```
1. 점심시간, 모바일로 Todo 앱 열기
2. 필터: "높음" 우선순위만 표시
3. 오전에 완료한 것들을 체크박스로 마크
4. 완료된 항목들이 흐린 표시 또는 리스트 하단으로 이동
5. 남은 "높음" 우선순위들을 보면서 오후 일정 파악
6. 필요하면 새로운 요청을 바로 추가
```

**인사이트**
- 필터 기능으로 "지금 필요한 것"만 노출
- 완료 처리 시 시각적 피드백(색상 변화, 위치 변화 등)이 만족감을 줌
- 높은 우선순위에 집중할 수 있도록 설계

## 지은(대학생)의 시나리오

### S5: "교수 공지에서 과제 즉시 추가"

**Context**  
수업 후 또는 학교 포탈에서 교수님의 새 과제 공지를 확인했다.  
마감 기한이 일주일인데, 빠르게 할 일에 추가해야 한다.

**Flow**
```
1. 학교 포탈/메일에서 과제 공지 확인
2. 휴대폰으로 Todo 앱 열기
3. 입력 필드에 과제명 입력 (예: "○○학 개론 리포트 제출")
4. 카테고리 드롭다운에서 "○○학 개론" 선택
5. 우선순위를 "중간" 또는 "높음" 선택 (마감일까지 고려)
6. Enter → 추가 완료
7. 나중에 카테고리별 필터링으로 모든 과제를 한눈에 볼 수 있음
```

**인사이트**
- 카테고리를 **빠르게 선택**할 수 있어야 함 (사전에 만들어둔 카테고리)
- 우선순위로 긴급도를 표시
- 추가 후 전체 리스트에서 카테고리별로 구분되어 보임

---

### S6: "저녁에 오늘 하루 정리"

**Context**  
저녁 9시, 침대에서 모바일로 오늘 하루를 정리한다.  
완료한 과제, 공부한 것들을 체크하고 내일 준비를 한다.

**Flow**
```
1. 침대에 누워 모바일로 Todo 앱 열기
2. "활성" 필터 클릭 (미완료 항목만 보기)
3. 오늘 완료한 것들을 체크박스로 마크
4. 각 항목이 흐린 색으로 변하면서 만족감 느낌
5. 완료된 항목들을 "완료" 필터로 보면서 오늘 한 일 정리
6. "활성" 필터로 돌아와 내일 할 일 확인
7. 필요하면 내일 할 일의 우선순위 조정
```

**인사이트**
- 상태 필터(전체/활성/완료)로 **마음가짐 전환** 가능
- 완료 처리 시 명확한 시각적 피드백(색상, 위치)이 만족감을 줌
- 하루를 심리적으로 "마무리"하는 느낌

---

### S7: "시험 2주 전 장기 계획 수립"

**Context**  
주말 오후(2주 뒤 시험 때문에). 노트북에서 한 주 공부 계획을 체계적으로 세운다.

**Flow**
```
1. 데스크톱에서 Todo 앱 열기 (넓은 화면에서 여유 있게)
2. "시험" 카테고리 하단에 과목별로 세부 카테고리 작성 고려
   또는 각 과제마다 우선순위 조정
3. 과목별 강의 내용 정리, 문제 풀이 등을 다시 정리
4. 각 항목에 우선순위 설정 ("시험에 자주 나오는 것"부터)
5. 필터로 "시험" 관련 항목만 보며 전체 일정 조감
6. 우선순위 순서대로 한 주 공부 계획 수립
```

**인사이트**
- **데스크톱에서** 넓은 화면으로 여유 있게 정보 입력
- 카테고리와 우선순위의 조합으로 체계적 관리
- 필터 기능으로 특정 영역(시험)만 보면서 집중

---

### S8: "카테고리별 진도 확인"

**Context**  
주중 저녁. "○○학 개론" 과목의 이번 주 진도를 확인하고 싶다.

**Flow**
```
1. 모바일로 Todo 앱 열기
2. 카테고리 필터에서 "○○학 개론" 선택
3. 해당 과목의 모든 과제/공부 항목이 보임
4. "활성" 필터로 미완료 항목 확인 → 이번 주 할 일
5. "완료" 필터로 이번 주 완료한 것 확인 → 진도 체크
6. 활성으로 돌아와서 남은 일들을 우선순위 순으로 봄
```

**인사이트**
- 카테고리 필터로 **특정 과목만 집중**할 수 있음
- 상태(활성/완료) + 카테고리 조합으로 "이 과목 이 주 진도"를 한눈에 파악
- 진도 관리의 만족감

## 겹치는 시나리오 (민준과 지은 공통)

### S9: "완료 처리의 만족감"

**Context**  
민준과 지은 모두, 일을 완료했을 때.

**Flow**
```
1. 할 일 완료
2. 앱을 열고 해당 항목의 체크박스 클릭
3. 항목이 흐린 색으로 변하거나 리스트 하단으로 이동
4. "완료한 일"이라는 시각적 피드백 확인
5. 만족감 느낌 → 다음 일로 넘어감
```

**인사이트**
- 완료 피드백이 명확해야 심리적 만족감을 줌
- 색상 변화, 위치 변화 등 **시각적 강화**
- 일을 체계적으로 처리하고 있다는 느낌

## 설계 원칙 정리

| 번호 | 원칙 | 관련 시나리오 | 설계 방향 |
|------|------|------------|---------|
| **P1** | 입력 필드가 가장 위에 있고, Enter만 눌러도 추가 가능 | S1, S2, S5 | UX: 메인 입력 최상단 배치 |
| **P2** | 기본값(카테고리="일반", 우선순위="중")으로 즉시 추가 | S1, S2 | 초기값 설정으로 빠른 추가 |
| **P3** | 우선순위 변경이 즉시 리스트 재정렬에 반영됨 | S3 | UI: 우선순위 순으로 자동 정렬 |
| **P4** | 우선순위와 카테고리로 시각적 강조 (색상, 배치) | S3, S4, S7 | 시각적: 높은 우선순위를 눈에 띄게 |
| **P5** | 카테고리 필터로 "관심 있는 일"만 빠르게 노출 | S4, S5, S8 | 필터링: 카테고리별 뷰 제공 |
| **P6** | 데스크톱과 모바일에서 동일한 기능 제공 | S7, S6 | 반응형: 모두 동등한 경험 |
| **P7** | 완료 처리 시 명확한 시각적 피드백 (색상, 위치 변화) | S4, S6, S9 | UI: 완료 상태를 명확하게 표시 |
| **P8** | 상태 필터(전체/활성/완료)로 마음가짐 전환 가능 | S6, S9 | 필터링: 상태별 뷰로 심리적 만족감 |

---

## 결론

**민준의 핵심:** 빠른 추가 + 우선순위 관리 → 시간에 쫓기는 직장인의 번개같은 손가락  
**지은의 핵심:** 체계적 분류 + 카테고리 필터 → 계획을 세우는 학생의 정리 습관  
**공통:** 완료 처리의 만족감 → 심리적 보상으로 지속적 사용 유도

설계는 "**빠르고 깔끔함**"을 중심으로, 모든 기능이 **한 화면에서 놓치지 않도록** 구성.
