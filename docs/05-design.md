# 디자인 문서 (05-design.md)

## 디자인 컨셉: 대시보드 스타일 (시안 C)

**철학:** "명랑하고 현대적. 진도와 할 일을 한눈에 파악하고, 완료할 때마다 느끼는 만족감."

**선택 이유:**
- 진도 시각화 탁월 (카테고리별 진도 한눈에)
- 정보 밀도와 심플함의 균형
- 완료 후 시각적 피드백이 강함
- 모바일에서도 스크롤로 모든 정보 접근 가능

## 색상 팔레트

### 주색상 (Primary)

```
그라데이션: #667eea → #764ba2 (보라색 → 진보라색)
- 사이드바, 강조, CTA 버튼에 사용
- 신현감 + 프리미엄 느낌
```

### 배경색 (Background)

```
그라데이션 배경: #f5f7fa → #c3cfe2 (연한 파란색 → 라벤더)
- 전체 앱 배경
- 연한 톤으로 콘텐츠를 돋보이게 함
- 밝고 명랑한 느낌
```

### 카드 배경

```
흰색: #ffffff
- 모든 카드(진도, 입력, 할 일)의 배경
- 깔끔하고 읽기 쉬움
```

### 텍스트 색상

```
제목: #333333 (진회색)
본문: #666666 (중간 회색)
보조: #999999 (연한 회색)
```

### 우선순위별 색상

```
높음 (High):   #FF6B6B (빨간색) - 긴급감
중간 (Medium): #FFB84D (주황색) - 주의
낮음 (Low):    #4ECDC4 (청록색) - 여유

- 테스트: 색약자도 구분 가능 (모양으로도 구분)
```

### 상태 색상

```
완료: #4ECDC4 (청록색, 부드럽고 긍정적)
진행중: #667eea (보라색, 활동적)
에러: #FF6B6B (빨간색, 주의)
```

### 그림자 (Shadow)

```
카드 그림자: 0 2px 8px rgba(0,0,0,0.08) (매우 부드러움)
호버 그림자: 0 4px 12px rgba(0,0,0,0.12) (약간 더 진함)
```

## 레이아웃 및 간격

### 여백 (Padding/Margin)

```
대형 (24px): 섹션 간 간격
중형 (16px): 카드 내부, 컴포넌트 간
소형 (12px): 내부 요소, 세부사항
미니 (8px): 인라인 요소, 아이콘

원칙: 8px 그리드 시스템 사용
```

### 모서리 (Border Radius)

```
큰 카드: 12px (진도, 입력, 할 일 카드)
버튼: 8px (추가, 선택 등)
입력: 8px (텍스트 필드)
```

## 컴포넌트 명세

### 1. 진도 카드 (Progress Card)

```
레이아웃:
┌─────────────────┐
│ 업무            │
│                 │
│       60%       │
│ ████░░░░░░      │
└─────────────────┘

구성:
- 카테고리명 (상단, 12px, 굵기 600)
- 백분율 (중앙, 28px, 굵기 700, 그라데이션)
- 진도 바 (하단, 높이 6px, 그라데이션)

배경: 흰색 (#ffffff)
그림자: 0 2px 8px rgba(0,0,0,0.08)
모서리: 12px

반응형:
- 데스크톱(1024px+): 3~4개 카드 가로 배열
- 태블릿(640px~1024px): 2개 카드 가로 배열
- 모바일(320px~640px): 1개 카드 세로 배열
```

### 2. 입력 카드 (Input Card)

```
레이아웃:
┌──────────────────────────────────────┐
│ [텍스트 입력        ] [추가]          │
└──────────────────────────────────────┘

구성:
- 텍스트 입력: flex 1, 높이 36px, 보더 1px #e5e5e5
- 추가 버튼: 배경 그라데이션, 흰색 텍스트, 높이 36px

배경: 흰색 (#ffffff)
그림자: 0 2px 8px rgba(0,0,0,0.08)
모서리: 12px
패딩: 16px
갭: 12px
```

### 3. 할 일 카드 (Task Card)

```
레이아웃:
┌─────────────────────────────────────┐
│ ☐ 보고서 초안 작성                  │
│   업무 • 높음                       │
└─────────────────────────────────────┘

구성:
- 체크박스 (20x20px, 모서리 6px)
- 할 일 제목 (14px, 500 굵기)
- 메타정보 (12px, 회색):
  - 카테고리 (배경색 #f0f0f0)
  - 우선순위 (텍스트 색상, 높음/중간/낮음)

보더-상단: 3px 실선 (우선순위에 따라 색상 변경)
  - 높음: #FF6B6B
  - 중간: #FFB84D
  - 낮음: #4ECDC4

배경: 흰색 (#ffffff)
그림자: 0 2px 8px rgba(0,0,0,0.08)
호버 시: 0 4px 12px rgba(0,0,0,0.12), translateY(-2px)
모서리: 12px
패딩: 16px

완료 상태:
- 체크박스에 체크 표시
- 텍스트 색상: #999 (진회색)
- 텍스트 장식: line-through
```

### 4. 필터 탭

```
구성:
[필터] [활성] [완료]

기본 상태:
- 배경: 투명
- 텍스트: #666
- 보더-하단: 2px 투명

활성 상태:
- 텍스트: #007AFF (파란색)
- 보더-하단: 2px #007AFF
- 굵기: 600

높이: 36px
패딩: 0 12px
폰트: 14px
```

## 인터랙션

### 클릭/터치 피드백

```
1. 입력 필드 포커스
   → 보더 색상 변경 (파란색)
   → 배경 약간 밝아짐

2. 할 일 카드 호버 (데스크톱)
   → 그림자 강화: 0 4px 12px rgba(0,0,0,0.12)
   → Y축 -2px 이동 (들뜬 느낌)

3. 체크박스 클릭
   → 즉시 체크 표시
   → 카드 배경 약간 밝아짐
   → (E1 기능) 작은 애니메이션 (폭죽 이모지 깜박임)

4. 우선순위 변경
   → (E2 기능) 카드가 부드럽게 재정렬
   → 보더 색상이 새로운 우선순위로 변경

5. 필터 탭 클릭
   → 탭 활성화 (보더-하단 강조)
   → 리스트 새로고침 (부드러운 페이드)
```

### 애니메이션

```
카드 진입: 페이드인 (300ms)
카드 이동: 부드러운 슬라이드 (200ms)
체크: 폭죽 또는 체크마크 애니메이션 (300ms)
호버: 그림자/위치 변경 (150ms)

전체 타이밍: ease-out (자연스러운 감속)
```

## 접근성 (Accessibility)

### 색상 대비

```
- 텍스트 vs 배경: 4.5:1 이상 (WCAG AA 표준)
- 우선순위 색상: 색각이상자도 구분 가능하도록
  → 색상만 아니라 텍스트("높음", "중간", "낮음")로도 구분
```

### 키보드 네비게이션

```
Tab: 입력 → 버튼 → 체크박스 → 필터 탭 순서
Enter: 추가 버튼 활성화
Space: 체크박스 토글
```

### 스크린 리더

```
- 각 카드: role="article" 또는 <article>
- 체크박스: <input type="checkbox" aria-label="...">
- 필터 탭: role="tablist" / role="tab" / aria-selected
- 진도 바: aria-valuenow, aria-valuemin, aria-valuemax
```

### 포커스 표시

```
모든 포커스 가능 요소:
- outline: 2px solid #007AFF
- outline-offset: 2px
```
